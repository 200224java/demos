CSS - Cascading Style Sheets

So now we have a deep understanding of HTML, but the last page still looked pretty ugly. The browser applies some default styling with certain tags, like bolding <strong> tags, but we were still looking at a very basic document - no colors, no other fonts, no spacing. So how do we style the page to make it look nicer? We use CSS!

As you read through this page, feel free to stop and experiment with writing CSS here - the best way to learn is by trying it out yourself!

CSS stands for Cascading Style Sheets. It's a way of styling our HTML markup using rules that we define. There are three ways to include CSS in our HTML file:

  • inline styling
  • using internal stylesheet(s)
  • using external stylesheet(s)

On this page, we'll use inline styles and an internal stylesheet, but the best practice is to use an external stylesheet because it allows for separation of concerns - basically, your HTML and CSS will be separated, so that changing a rule in one place will apply to every HTML page that uses that stylesheet.

Inline Styling

Inline styling can be performed on any HTML element using the style attribute. Within the quotation marks defining the value of the style attribute, we define a CSS rule which tells the browser what styling to apply. CSS rules are key/value pairs, separated by a colon (:). There are many CSS rules we can apply, but let's start with the color attribute, which will change the color of the font.

When we write this CSS within the style attribute of the span tag: 

 
    <span style="color: blue">this text appears blue</span>

This will be the result: this text appears blue

Now let's add another rule, changing the background color for the element: 

 
    <span style="color: blue; background-color: orange">this text appears blue with an orange background</span>

This will be the result: this text appears blue with an orange bg.

Note that CSS rules are separated by semi-colons (;).

Internal Stylesheet

If you look in the head of this document, you'll notice a style tag - within here we can write a CSS stylesheet that will apply to the entire document. A stylesheet is a set of CSS selectors with associated rules for styling our HTML.

A CSS selector is how we select specific HTML elements that we want to style. After the selector we place curly braces - {} - and within the curly braces we write the CSS rules which only apply to elements selected by that CSS selector. The simplest selector is just the name of the HTML tag you want to style - note that this will affect all elements with that tag name. An sample excerpt from this page's internal stylesheet is shown below: 

    body {
        max-width: 800px;
        margin: auto;
        font-family: Helvetica, Arial; /* multiple values! */
      }

This CSS restricts the width of the body to be 800 pixels at maximum and sets the margin to be automatically calculated, which centers the element within its parent element (which is the entire html element - the whole page). That CSS is what makes this page appear as a single column of text centered in the middle of the page.

The final rule specifies the font family - multiple values are provided and separated by a comma, so that if the browser does not support a specific font, a fallback value can be used instead. Here we prefer to use Helvetica, but will fallback to Arial if for some reason the browser doesn't support Helvetica.

Also, note that CSS comments start with /* and end with */.

Selectors

Simple Selectors

We just saw an example of a simple selector - using the tag name to identify the element. As another example, we can select all the divs on our page and make their fonts red like so: 

          div { color: red; }

Class and ID Selectors

What if we don't want to select all elements with the same tag name, though? We can be more specific about a subset of elements using the class and id selectors.

The class selector is denoted by a period followed by the class name (.classname) and selects all elements whose class attribute's value contains the specified value - which you define. You can define styling for a specific class and then give an element that class to apply the styling. For example, the following CSS styles set the height and width of all elements of the jumbo class. 

            .jumbo { height: 1000px; width: 2000px; }

You can also combine tag selectors with class selectors to get more specific. To refine our CSS above, we can apply the styling only to divs with the jumbo class: 

            div.jumbo { height: 1000px; width: 2000px; }

id is an attribute that can be applied to any element and uniquely identifies it - therefore, whatever value is provided cannot be used anywhere else within the HTML document.

The id CSS selector is denoted by a pound sign (or hashtag, if you prefer), followed by the id: #uniqueIdHere. For example, let's say we have an HTML element: <span id="first">I am first!</span>. We can target this element and style the font size with the following CSS: 

          #first { font-size: 32pt; }

We can combine id and other selectors, but there really shouldn't be any need to - since the id should only apply to a single element anyways.

Pseudo-class and pseudo-element selectors

A pseudo-class is a specific keyword at the end of a selector used to specify that you want to style the selected element but only when it is in a certain state - for example, when the user is hovering the mouse over the element, or when a checkbox is disabled or checked. The syntax is to use a colon (:) to precede the pseudo-class. Here's a short list of pseudo-class selectors (there are many more):

  • :active
  • :visited
  • :checked
  • :disabled
  • :first
  • :nth-child()
  • :hover

For example, the following CSS will make a button become green when the user hovers over it with the mouse: 

          button:hover { background-color: green; }

Pseudo-elements are similar to pseudo-classes but are prepended by two colons. Some common pseudo-elements are:

  • ::before
  • ::after
  • ::first-line
  • ::first-letter

Child and Descendant Selectors

The previous selectors are fine, but what if you only want to select elements that are nested within other elements - like selecting all divs within other divs? For that, we need descendant selectors.

There are two different ways of selecting nested elements - descendant and direct descendant (or child). To select elements that are direct descendants of other elements, use the > operator. For example, to select all span elements that are direct children of any div: 

          div > span { ... }

This works fine if we have this HTML structure: 

          <div>
            <span>some text</span>
          </div>

But what if we have a p wrapping around the span? 

          <div>
            <p>
              <span>some text</span>
            </p>
          </div>

To select the span this time, we need to use the general descendant selector, which is simply an empty space. So we take out the > and now, no matter how deep the span is nested inside the div, we can still style it with this CSS: 

          div span { ... }

Other Selectors

Sibling selectors
There are two types of sibling selectors - general and adjacent. General sibling selectors are denoted by ~ and adjacent sibling selectors are denoted by +. Here's an example: 
        p ~ ul { ... } /* general - select any ul elements that follow a p element and share the same parent  */
        p + ul { ... } /* adjacent - select any ul elements that follow directly after a p element */
Attribute selectors

You can select elements by attribute using the square brackets []. Additionally, use certain special characters to match patterns:

  • ~ - value must match within a space-separated list of words
  • ^ - starts with the value
  • $ - ends with the value
  • * - contains the substring value

Example of these kinds of selectors: 

      a[href] { ... } /* selects all anchor tags with the href attribute, no matter the value */
      a[href="http://www.google.com"] { ... } /* selects all anchor tags with the href attribute that exactly match the URL */
      a[href~="http://www.google.com"] { ... } /* selects all anchor tags with the href attribute that has the URL as one of its values */
      a[href^="https"] { ... } /* selects all anchor tags with the href attribute that start with "https" */
      a[href$=".com"] { ... } /* selects all anchor tags with the href attribute that end with ".com" */
      a[href*="google"] { ... } /* selects all anchor tags with the href attribute that contain "google" somewhere in the URL */
Multiple selectors

Combine multiple selectors with commas: 

            div, p { color: pink; } /* make the text of all divs and paragraphs pink  */
            .jumbo, #first { background-color: orange; } /* make all elements with the class jumbo or id first have an orange background */

Wildcards

The * wildcard character can be used to specify "all" elements, and can be used in any context. Here are a few examples: 

        * { border: 1px solid black; } /* gives ALL elements on a page a thin border. try it out on this page! */
        div > * { color: green; } /* gives green text to all elements that are direct descendants of divs */

CSS Box Model

The CSS box model is the way that HTML elements are defined - there are actually 4 different concentric boxes, which are - from innermost to outermost:

  • Content - this contains the actual text or image
  • Padding - space between content and border
  • Border - space between padding and margin - its width, style, and color properties may be set
  • Margin - usually invisible space between the end of one element and the start of another

The padding, border, and margin can all be styled, on each side of the box or alltogether. For more information on this, look here. Here's a few examples: 

          .boxExample {
            width: 200px;
            height: 20px; /* width and height set the size of the content box */
            padding: 5px 1px 8px 20px; /* TRBL - top, right, bottom, left - respectively */
            border: 2px solid green; /* set the border on every side to be 2px wide, solid line, and green background color */
            /* alternatively, we could use apply these styles individually:
            border-width: 2px;
            border-style: solid;
            border-color: green;
            */
            margin-top: 10px; /* set the margin on the top to be 10 pixels */
            /* alternatively, we could use shorthand if we want to apply a margin on all sides:
            margin: 10px; */
          }

Units

There are several different types of values we can provide for styles, and each has their own units we can use. These are specified below:

  • Sizes
    • Absolute
      • px - pixels
      • mm - millimeters
      • cm - centimeters
      • in - inches
      • pt - points (1/72 of an inch)
      • pc - picas (12 points)
    • Relative
      • % - percentage relative to parent element's width
      • em - 1em is the same as the font size of the current element
      • rem - stands for "root em" and works the same except refers to the base font size
      • vh, vw - these are 1/100th of the height and width of the viewport, respectively
  • Colors
    • keywords - like red, blue, even specific ones like aquamarine
    • hex value - # sign followed by six hex numbers (0-F) with each pair representing rgb values for up to 256 different values of each
    • rgb(a) - a function to specify values for red, green, and blue channels, use like: rgb(224,176,255). Optional alpha (transparency) value.
    • hsl(a) - a function to specify hue, saturation, and lightness values to define a color, use like: hsl(240,100%,50%). Optional alpha (transparency) value.
  • Functions
    • rgb()
    • hsl()
    • rotate()
    • translate()
    • calc()
    • url()
